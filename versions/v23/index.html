EasyDo 365 — v23 (Role-distinct heroes)

The first version that drops the "one hero for everyone" mistake. Six permanent roles from the backend's authz.ts catalog, each with a distinctly-shaped Home hero: Owner sees an attention queue, HR Admin sees an onboarding pipeline, Manager sees branch pulse, Team Lead sees team status, Employee keeps Punch, Member gets an onboarding checklist. SF-Symbol-style icons throughout.

The demo defaults to Owner so the first impression is "decision queue for a multi-branch business," not "tap to clock in." Use the role switcher on the right to scrub between the six heroes — same chassis, six different products.
